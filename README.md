<h1>Общее описание:</h1>
Dependency Visualizer - это CLI-приложение для анализа и визуализации графа зависимостей пакетов в репозиториях Ubuntu/Debian. Приложение поддерживает как работу с реальными репозиториями, так и тестовыми данными.

<h1>Основные возможности:</h1>
<p>- Анализ прямых и транзитивных зависимостей пакетов</p>
<p>- Построение графа зависимостей с помощью DFS алгоритма</p>
<p>- Ограничение глубины анализа</p>
<p>- Фильтрация пакетов по имени</p>
<p>- Обнаружение и обработка циклических зависимостей</p>
<p>- Визуализация в формате ASCII-дерева</p>
<p>- Поддержка тестового режима с пакетами-заглушками</p>

<h1>Описание всех функций и настроек:</h1>
<p>- main.py: Основной модуль приложения</p>
<p>- parser.py: Парсинг командной строки</p>
<p>- config.py: Конфигурация и валидация</p>
<p>- errors.py: Пользовательские исключения</p>
<p>- apt_parser.py: Парсер APT репозиториев</p>
<p>- graph_builder.py: Построитель графа зависимостей</p>
<p>- test_repository.py: Работа с тестовыми репозиториями</p>
<p>- test_repo.txt: Пример тестового репозитория</p>

<h1>Модули и их функции:</h1>
<h2>main.py</h2>
<p>- DependencyVisualizer: основной класс приложения</p>
<p>- analyze_dependencies(): основной метод анализа</p>
<p>- _display_dependency_info(): вывод информации о графе</p>
<p>- _display_ascii_tree(): визуализация дерева зависимостей</p>

<h2>parser.py</h2>
<p>- CommandLineParser: парсинг аргументов командной строки</p>
<p>- Поддержка всех необходимых флагов и параметров</p>

<h2>config.py</h2>
<p>- Config: хранение и валидация конфигурации</p>
<p>- Валидация URL и параметров</p>

<h2>apt_parser.py</h2>
<p>- APTParser: работа с APT репозиториями</p>
<p>- get_package_dependencies(): получение зависимостей пакета</p>

<h2>graph_builder.py</h2>
<p>- GraphBuilder: построение графа зависимостей</p>
<p>- build_dependency_graph(): основной метод построения</p>
<p>- _dfs(): рекурсивный DFS обход</p>
<p>- Обнаружение циклических зависимостей</p>

<h2>test_repository.py</h2>
<p>- TestRepository: работа с тестовыми репозиториями</p>
<p>- Поддержка пакетов, обозначенных большими латинскими буквами</p>

<h1>Флаги:</h1>
<p>- Имя пакета</p>
<p>- URL репозитория</p>
<p>- Режим тестирования: "--test-repo-mode"</p>
<p>- Файл вывода: "--output"</p>
<p>- ASCII-дерево: "--ascii-tree"</p>
<p>- Макс. глубина: "--max-depth"</p>
<p>- Фильтр "--filter"</p>

<h1>Команды для сборки и запуска тестов</h1>

<h2>Требования:</h2>
<p>- Python 3.6+</p>
<p>- Доступ к интернету (для работы с реальными репозиториями)</p>

<h2>Вариации запуска приложения:</h2>

```bash
# Базовая команда
python main.py <package_name> <repository_url> [options]

# С реальным репозиторием + аски дерево
python main.py python3 http://ftp.de.debian.org/debian/ --ascii-tree

# С тестовым репозиторием + аски дерево
python main.py A test_repo.txt --test-repo-mode --ascii-tree

# С ограничением глубины = 2
python main.py A test_repo.txt --test-repo-mode --max-depth 2

# С фильтрацией пакетов = "C"
python main.py A test_repo.txt --test-repo-mode --filter "C"
```

<h1>Особенности реализации:</h1>
<p>- Настраиваемое CLI - приложение</p>
<p>- Формат пакетов: Ubuntu (apt)</p>
<p>- Извлечение информации пакета через URL репозиторий</p>
<p>- DFS с рекурсией - граф строится с помощью глубинного поиска</p>
<p>- Обработка циклов - обнаружение и маркировка циклических зависимостей</p>
<p>- Фильтрация - исключение пакетов по подстроке на этапе построения графа</p>
<p>- Ограничение глубины - остановка рекурсии при достижении максимальной глубины</p>
<p>- Тестовый режим - работа с упрощенными пакетами для демонстрации</p>
